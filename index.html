Logic Explanation

The application stores loot in one array named lootItems. Each time the user clicks Add Loot, the program validates the form data first, then saves the loot as an object with a name and value. A traditional for loop in calculateTotalLoot() adds the values from every object in the array. Another traditional for loop in renderLoot() rebuilds the visible loot list on the page. Conditional logic appears in addLoot() and splitLoot() to check for empty names, empty values, invalid numbers, negative values, missing party size, party size below 1, and the case where no loot has been entered before splitting.
